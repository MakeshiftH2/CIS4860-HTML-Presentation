Project Charter

Showcasing Content to X Audience

A modernized charter to create, distribute, and iterate on game and digital-art content with an audience-first strategy.

Mission

Build a reusable content creation environment and showcase pipeline that reliably delivers high‑quality maps, 3D assets, and digital art to a defined audience across multiple platforms—then learns from the feedback.

Vision

A portfolio‑ready body of work and a lightweight analytics loop that aligns creation with audience needs and drives continuous improvement.

Success Criteria
  • Ship 2–3 maps, a 5–8 item 3D asset pack, and 3–5 digital art pieces.
  • Publish to ≥3 platforms targeting X Audience segments.
  • Collect ≥30 audience feedback responses; document a repeatable procedure.
  • ≥80% of DoD checks passed; roadmap milestones met.

Scope

In Scope

  • Create core content: 2–3 game map layouts, 5–8 item 3D asset pack, 3–5 digital art pieces.
  • Audience targeting: define X Audience segments and value propositions.
  • Multi‑platform distribution: publish across appropriate channels (e.g., itch.io/portfolio, ArtStation/Behance, Instagram/YouTube Shorts, Reddit/Discord).
  • Feedback collection framework: survey links/QR codes, comment prompts, UTM‑tagged links, simple analytics dashboard.
  • Final deliverable: a documented, repeatable Audience Feedback Procedure with templates and measurement plan.

Out of Scope

  • Full commercial game production; VR/AR builds.
  • Paid advertising buys (unless separately approved).
  • Advanced data warehousing/ETL beyond lightweight analytics.
  • Monetization/branding rollout (moved to future roadmap).
Note
Scope caps are enforced via weekly sprint limits and DoD checklists to prevent creep.

Deliverables & Definition of Done
  1. Content Creation Environment — repo + folder standards; templates (DoD, change log, release notes); naming/tagging scheme.
  2. 2–3 Game Map Layouts — blockouts + exports (FBX/GLB); playtest screenshots or short clips; release notes.
  3. 5–8 Item 3D Asset Pack — clean topology, UVs, license file, preview renders, download package.
  4. 3–5 Digital Art Pieces — PNG/JPG exports (2 resolutions), process notes, platform‑ready thumbnails.
  5. Portfolio & Multi‑Platform Publishing — posts on ≥3 platforms (e.g., itch.io/portfolio, ArtStation/Behance, Instagram/YouTube Shorts) with consistent branding and CTAs.
  6. Audience Feedback Procedure (Final Deliverable) — step‑by‑step guide, templates (survey, comment‑coding sheet), analytics checklist, decision rules for iteration.
DoD
Each deliverable includes evidence links, version tag (e.g., env1_mapA_v0.3_graybox), and passes peer/instructor review per rubric.
Final Deliverable — Procedure for Obtaining Audience Feedback

Collection Channels

  • Survey (Google Forms): QR + short link on every post; 6–8 concise questions.
  • Comments (platform‑native): prompt with 1 simple question per post.
  • Direct Messages (Discord/Reddit): invite opt‑in testers via pinned message.
  • Analytics: UTM‑tagged links to portfolio/itch page; basic view/like/click metrics.

Templates

  • Survey link: https://forms.gle/your‑id + QR PNG in \assets\feedback
  • Comment prompt: “What part of this map feels most fun or confusing?”
  • Tagging sheet: columns = Platform, Post, Comment, Tag, Sentiment (+/‑/0), Action

Processing & Decisions

  1. Collect responses weekly (Fri).
  2. Code comments by themes (visual style, readability, difficulty, performance).
  3. Summarize: top 3 insights, top 3 fixes, one quick experiment.
  4. Update backlog; schedule fixes in next sprint; post follow‑up.
Privacy
Use anonymous collection unless explicit consent is provided. Do not collect sensitive personal data.

Risks & Mitigations
  • Scope creep → Sprint caps; DoD gates; backlog parking lot.
  • Time compression → Timeboxes; weekday micro‑tasks; weekend integration blocks.
  • Platform friction → Early test posts; export presets; thumbnail/caption templates.
  • Low feedback volume → Clear CTAs; incentives (shout‑outs); cross‑posting; QR at campus events.
